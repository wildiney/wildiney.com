Vibe coding ou Lucky coding?

A IA democratizou a execução. Mas não democratizou o julgamento. Lucky code é só achismo com interface moderna.

vibe coding inteligência artificial engenharia ágil decisão desenvolvimento de produto karpathy

Vibe coding virou buzzword. E como toda buzzword, perdeu precisão no caminho. Mas a história começa antes disso.

Quando os modelos de linguagem explodiram, surgiu o prompt engineering: uma abordagem metódica para guiar a IA com clareza, contexto e intenção. Chegou a ser considerada uma das profissões mais quentes de 2023. Dois anos depois, estava praticamente obsoleta.

O mercado não evoluiu. Ele trocou método por vibes.

Em fevereiro de 2025, Andrej Karpathy, cofundador da OpenAI, cunhou o termo vibe coding: "dar as vibes para a IA, abraçar os exponenciais e esquecer que o código existe." A ideia era para projetos rápidos e descartáveis. O mercado transformou isso em metodologia de produção.

E criou dois perfis que quase ninguém nomeia.

Existe o que eu chamo de lucky code: você escreve um prompt, torce para a sorte te ajudar e chama o resultado de produtividade. Funciona às vezes. Por acaso. E quando para de funcionar, ninguém sabe exatamente por quê, então o próximo passo é tentar outro prompt aleatório e torcer de novo.

E existe o uso profissional e estruturado: antes de qualquer prompt, você cria um PRD, Product Requirements Document, que define o propósito, as funcionalidades e o comportamento do produto. É ele que orienta a IA dentro de um raciocínio claro. Quando algo precisa ser ajustado, você atualiza os parâmetros. Não reza um novo prompt esperando um milagre diferente.

Curiosamente, em fevereiro de 2026, exatamente um ano depois, o próprio Karpathy voltou com um novo termo para esse uso profissional: agentic engineering. "Agentic porque você orquestra agentes, engineering para enfatizar que existe arte, ciência e expertise nisso."

O mercado foi de prompt engineering para vibe coding e voltou para engenharia. Com outro nome.

Mas há um problema que antecede qualquer termo.

Muita gente nem chega a ter uma ideia própria. Não pesquisa o problema. Não valida se existe valor real no que quer construir. Não testa com usuários reais. E então manda a IA executar, usando como referência aquele benchmarking tão celebrado nas apresentações corporativas, que na prática significa copiar o concorrente pixel a pixel.

O produto original que você está replicando já tem um time inteiro trabalhando nos problemas que você não consegue ver na interface. Você está copiando a fachada e herdando todos os defeitos estruturais que eles já estão correndo para corrigir.

A IA democratizou a execução. Mas não democratizou o julgamento.

Lucky code é só achismo com interface moderna.
